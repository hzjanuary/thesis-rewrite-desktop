# Greenfield Project: Native Desktop App for AI Thesis Rewriter

## 1. Project Overview
Act as an expert Systems Programmer (Rust) and Full-stack Developer (Tauri + React). We are building a brand-new Native Desktop Application from scratch. 
The app is an offline "AI Writing Assistant" that runs a quantized Small Language Model (SLM) natively on the user's machine to rewrite and polish academic texts. 

We are bypassing browser WebGPU memory limits (`std::bad_alloc`) by building this strictly as a Desktop App using **Tauri v2**. This allows us to allocate system RAM/VRAM directly via OS-level native execution.

## 2. Tech Stack Setup (Create from scratch)
- **App Framework:** Tauri v2.
- **Frontend UI:** React + Vite + TailwindCSS + TypeScript. 
- **Backend (AI Inference):** Rust.
- **AI Core:** `llama.cpp` Rust bindings (e.g., `llama_cpp_2` or equivalent crate) configured to load `.gguf` models.
- **Target Model:** Qwen2.5-1.5B-Instruct (4-bit quantized GGUF, approx 1GB).

## 3. Core Architecture & Requirements

### A. OS-Level Model Management (Rust)
- The app must not bundle the 1GB model in the installer. 
- On first launch, the Rust backend must download the `.gguf` model file from Hugging Face into a persistent OS-native application data folder (e.g., `AppData/Local/AppName` on Windows or `~/Library/Application Support/AppName` on macOS).
- **Frontend-Backend Sync:** Provide a Tauri command to start the download and use Tauri Events to stream the download progress (0-100%) back to the React UI in real-time.

### B. Native Inference Engine (Rust + llama.cpp)
- Once downloaded, Rust will load the `.gguf` model into system memory/VRAM using `llama.cpp`. This guarantees optimal CPU/iGPU fallback and bypasses all browser sandboxes.
- **Token Streaming:** Create an `invoke("rewrite_text", { text, style })` command. As the Rust backend generates the response, it MUST stream the output token-by-token back to the React UI via Tauri Events for a smooth typing effect.

### C. UI/UX Design (Frontend)
- Implement a modern, minimalist, split-screen layout.
- **First Launch Screen:** A clean "Setup" screen showing a progress bar for the model download.
- **Main Editor Screen:**
  - Left Panel: Textarea for user input + Dropdown to select writing style (Academic, Concise, Professional).
  - Right Panel: Output area displaying the streaming tokens natively generated by the Rust backend.
- **Theme:** Dark mode by default, utilizing clean typography and soft borders.
